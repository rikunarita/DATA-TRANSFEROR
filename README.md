# DATA-TRANSFEROR

DATA-TRANSFEROR は、最先端のインフラ経営をシミュレートするために開発された、ハイエンド・インフラ・タイクーンです。  
演算リソースの最適化、高度な人事戦略、そして冷酷な財務判断が、あなたの企業の時価総額を決定します。

## **🏛️ プロジェクトの核心 (Project Core)**

このシミュレーターは、現代のクラウドコンピューティングとデータセンター運営の複雑性を、美しいUIと厳密な数学的ロジックで表現したものです。  
プレイヤーはCEOとして、単なる「数字の増加」ではなく、\*\*「持続可能なスケーラビリティの確保」\*\*という究極の課題に挑みます。

### **💎 システムの4大柱**

1. **Dynamic Scaling (インフラ)**: CPU/GPUの物理的限界と冷却効率のバランスを調整。  
2. **Human Capital (人事)**: 従業員のスキルセットが演算効率や収益率に直結するリアルな労働市場。  
3. **Strategic R\&D (研究開発)**: 研究ポイント（DP）を消費し、業界のパワーバランスを塗り替える技術革新。  
4. **Capital Market (金融)**: 融資によるレバレッジ経営と、競合他社の株式保有によるポートフォリオ運用。

## **🧠 経営戦略ガイド：極限まで利益を出すコツ (Master Strategist's Guide)**

DATA-TRANSFERORの世界で「兆」を超える資産を築くための、開発者推奨の攻略アルゴリズムです。

### **1\. 序盤の「レバレッジ・スタート」**

* **戦略**: 開始直後に少額の「Bank Loan（融資）」を引き、**GPU**に集中投資してください。  
* **理由**: 初期の演算能力不足による「Latency（遅延）」は、企業信用度（Reputation）を致命的に下げます。借金をしてでも、まずインフラを「流入量以上」に整えることが、複利成長への第一歩です。

### **2\. 人事構成の黄金比 (The Staff Mix)**

* むやみに雇うのは厳禁。以下の優先順位を推奨します：  
  1. **SysAdmin**: ハードウェアの劣化を防ぎ、予期せぬメンテナンスコストを最小化します。  
  2. **Sales Exec**: 収益単価を上げるため、利益率を追求するなら必須です。  
  3. **DevOps**: 中盤以降、データ量が増えた際のレイテンシ削減に劇的な効果を発揮します。  
* **注意**: 「解雇」には多額の退職金が必要です。不況時に一気にクビを切ると、キャッシュフローがショートしてゲームオーバーになります。

### **3\. R\&D（研究開発）の優先ルート**

* 最優先は **「Auto-Scaling」** です。保守費を15%削減する効果は、長期的に見ると数百万ドルの利益差を生みます。  
* 次に **「Quantum Compression」**。スループットを上げつつ、既存のハードウェア寿命を延ばすことができます。

### **4\. 株式投資の「ヘッジ」**

* 自社のインフラが安定し、資金に余裕ができたら、ボラティリティ（価格変動）の激しい **「Nevada GPU (nvda)」** の株を安値で仕込んでください。  
* インフラ経営の収益を、このハイリスク・ハイリターンな投資に回すことで、資産増加速度が加速度的に上昇します。

## **🛠️ 技術仕様 (Technical Specifications)**

* **Frontend Framework**: React v18+  
* **Styling**: Tailwind CSS (JIT Engine)  
* **State Loop**: 100ms Tick-based Simulation (10FPS Logic)  
* **Persistence**: IndexedDB による完全自動セーブ（500ティック毎）  
* **Mathematical Icons**: Lucide-React による直感的なメタファー

### **🧮 収益計算式 (Profit Logic)**

$$NetProfit \= (ProcessedData \\times UnitPrice) \- (Salaries \+ Maintenance \+ Interest)$$

※ processedDataは min(ComputeCap, QueueSize) によって決定され、インフラの限界が収益の限界となります。
